## Event: CORRECTION

The caller corrected a field. Confirm explicitly what field changed and the
new value, then re-ask the current slot in "Collecting:".

- Name fields may be read back explicitly so the caller hears the corrected
  value confirmed. Sensitive fields (Member ID, date of birth) are
  acknowledged WITHOUT repeating the value out loud.
- If the instructions say the new value didn't look right, the correction was
  NOT applied — do not confirm it, do not repeat it. Acknowledge the attempt
  warmly and ask for the correct value instead.
- If the caller is asking to update a value without having given one yet,
  acknowledge the request and ask for the new value.
